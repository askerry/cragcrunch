Potential features

  1. climber x climb preference matrix
  2. unstructured text and other attributes (length, elevation) associated with each climb
  3. overall popularity

--> can I use this information to build a model that predicts the rating that an individual user gives to a specific climb?

Validation approach

I will use a variety of algorithms and feature sets to predict individual user ratings. To validate the model, I use leave-one-out cross validation. Can I predict the rating that an individual climber gives a climb based on their ratings of other climbs? Leave-on-out cross-validation has known limitations but is useful in the present context where there can be a very small number of exemplars for each individual climber. On each iteration, I train on all climbs but one, and predict the user's remaining climb.

read this: http://www.russpoldrack.org/2012/12/the-perils-of-leave-one-out.html

Feature selection

To be able to quickly incorporate new users who don't have a full climbing history, I want to identify a relatively simple, intuitive feature vector that can be used to provide predictions for a new user based on a few indicated preferences. So, I want to reduce this space of features. Many approaches to dimensionality reduction (i.e. PCA) reduce features into components that are linear combinations of the initial input space. However, the most intuitive inputs for the user are simple words like those contained in initial set of algorithmically selected features.

Forward stepwise iterative regression

Before diving into identifying dimensions, I want to explore the space of features a bit more and get a sense of which features explain the most unique variance across climbs. to do this, I do an iterative regression analysis (take individual feature with highest R2 and then iterate with the residuals) on the climb x feature matrix

1. for each user, take top N most important features on each iteration, and comput proportion of folds in which the feature was informative
2. compute one-sample T test across users to generate a tvalue reflecting how reliably informative a feature is
3. take top 100 features as final feature space
4. from this set of 100, used domain-knowledge to prune to 20 features to serve as a reduced feature space to use with new users

Feature pruning

We have 100 algorithmically selected features, but when we want to onboard new users, it won't make sense to ask them 100 questions to map them into this feature space. Also, not all the features make sense to ask as preference questions (i.e. "do you like great climbs" is an odd question). Thus, I reduce to smaller subset of 20 features that make sense to ask individual users and will use these as the reduced model space for new users. In contrast to the feature selection approaches used thus far, this final pruning from 100 to 20 features is performed using domain knowledge/product intuition.
